Detailed Description

String interning allocator built on top of arena_t.

Stores each unique string exactly once. Repeated calls to str_arena_intern with equal content return the same pointer, so pointer equality is sufficient for string comparison after interning.

Internally uses an arena_t for string storage and a hashmap_t keyed on FNV-1a uint64_t hashes for O(1) lookup. A strcmp fallback guards against the negligible probability of hash collisions.

typedef struct str_arena str_arena_t

String interning allocator.

Owns a backing bump allocator for storage and a hash index for O(1) lookup. Both are created and destroyed together with the str_arena.

The internal layout is opaque — always access through the str_arena API.

Interns a string, returning a stable pointer to its unique copy.

If s has been interned before, the previously stored pointer is returned and no allocation occurs. Otherwise s is copied into the backing arena and the new pointer is recorded in the index.

The returned pointer is valid until the string arena is destroyed.

Parameters
saThe string arena. Must not be NULL.
sThe string to intern. Must not be NULL.
errOptional error output. Populated on failure.
Returns
Stable pointer to the interned copy of s, or NULL on failure.
Note
Complexity: O(n) on the first call for a given string (n = length); O(1) on subsequent calls for the same content.
