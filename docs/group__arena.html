Detailed Description

General-purpose bump allocator with block chaining.

Memory is allocated from a chain of fixed-size blocks. Each allocation advances a pointer within the current block; when a block is exhausted a new one is appended to the chain. All memory is released at once when the arena is destroyed or reset.

Allocations are aligned to _Alignof(max_align_t) so the arena can safely hold any scalar or struct type, not just strings.

typedef struct arena arena_t

General-purpose bump allocator.

Dispenses memory from a chain of heap-allocated blocks. Allocations are O(1) and individually non-freeable. The entire arena is released in O(b) where b is the number of blocks allocated.

The internal layout is opaque — always access through the arena API.

Allocates a block of memory from the arena.

The returned pointer is aligned to _Alignof(max_align_t). If size exceeds the arena's block size, a dedicated oversized block is allocated to satisfy the request.

Parameters
arenaThe arena. Must not be NULL.
sizeNumber of bytes to allocate. Must be > 0.
errOptional error output. Populated on failure.
Returns
Pointer to the allocated memory, or NULL on failure.
Note
The returned memory is not zero-initialised.
Complexity: O(1) for allocations that fit in the current block; O(1) amortised overall (a new block is allocated only when the current one is exhausted).

◆ arena_reset()

void arena_reset ( arena_t *  arena,
error_t *  err 
)

Rewinds the arena without freeing its blocks.

Resets every block's used counter to zero, making all memory available for reuse. Previously returned pointers become invalid. Blocks are not freed, so subsequent allocations can reuse existing capacity without triggering new heap allocations.

Useful when the same arena is reused across repeated parses of different documents.

Parameters
arenaThe arena to reset. Must not be NULL.
errOptional error output. Populated on failure.
